## Vision

NAEOS aims to become the standard declarative operating system for engineering projects — a single source of truth that connects code, documentation, specifications, and AI-generated code in a coherent, verifiable framework.

## Current Focus

### Q3–Q4 2026

| Area | Status |
|------|--------|
| **NEIR v1.0 Specification** | 🚧 In Progress |
| **Plugin System** | 🚧 In Progress |
| **Multi-Project Workspaces** | 📋 Planned |
| **VS Code Extension** | 📋 Planned |
| **CI/CD Integrations** | 🚧 In Progress |
| **AI Context Provider API** | 📋 Planned |

## Completed Milestones

### Q1–Q2 2026

- ✅ **NAEOS CLI v0.x** — Core CLI with `init`, `run`, `generate`, `validate` commands
- ✅ **NEIR Format** — Declarative YAML-based engineering specification format
- ✅ **Module & Dependency Modeling** — Define project modules, services, and their dependencies
- ✅ **Code Generation** — Generate project scaffolding from NEIR models
- ✅ **OpenCode Integration** — AI-powered development workflow
- ✅ **Official Website** — NAEOS.dev documentation and community hub

## Future Plans

### 2027 and Beyond

- **NEIR v2.0** — Extended syntax with conditional modules, environment profiles, and inheritance
- **GUI Dashboard** — Visual project management interface
- **Enterprise Features** — RBAC, audit logging, SSO integration
- **Team Collaboration** — Real-time multi-user editing of NEIR models
- **Marketplace** — Community plugins, templates, and generators
- **Cloud Sync** — Optional cloud-based project synchronization
